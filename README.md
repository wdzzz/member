# member
会员周期管理系统
一个轻量、简洁的会员周期管理系统，基于 PHP + SQLite3 开发，专为群晖 Web Station 环境适配，可快速实现会员信息、周期、状态的基础管理。
🌟 项目简介
本系统由豆包撰写，核心定位是轻量化会员周期管理工具，无需复杂的数据库部署（依托 SQLite3 本地数据库），重点适配群晖 NAS 的 Web Station 环境，满足小型团队 / 个人对会员有效期、状态的基础管理需求。
🖥️ 适配环境
部署载体：群晖 NAS（需安装 Web Station 套件）
运行环境：PHP（推荐 7.4+，需启用 SQLite3 扩展）
数据库：SQLite3（无需独立部署，本地文件存储）
🚀 群晖部署步骤
1. 前置准备
登录群晖 DSM 后台，安装「Web Station」套件（套件中心搜索即可）；
将本项目完整文件上传至群晖的 Web 目录（默认路径：/volume1/web/member，可自定义）。
2. 关键配置（SQLite3 启用）
群晖 Web Station 中需确保 PHP 环境启用 SQLite3 扩展，步骤如下：
打开 Web Station → 点击左侧「脚本语言设置」；
选择对应运行的 PHP 版本（如 PHP 7.4）→ 点击「编辑」；
在「扩展」标签页中，勾选 pdo_sqlite 和 sqlite3 两个选项；
点击「保存」，重启 Web Station 使配置生效。
3. 启动系统
在浏览器中访问配置的地址：http://群晖IP:自定义端口/member（群晖 IP 为你的 NAS 局域网 IP，自定义端口为新增网页服务门户时设置的端口，member 为项目上传至 Web 目录的文件夹名，均根据实际部署配置调整）；
首次访问时，系统会自动加载init.php文件完成 SQLite3 数据库初始化：
自动生成数据库文件（无需手动创建）；
初始化管理员账号（默认账号：admin，默认密码：123456），为系统唯一管理账号；
访问后会先进入登录页面，输入上述默认账号密码即可登录系统；
建议登录后立即修改默认密码（系统内「账号设置」模块可修改），避免未授权访问；
系统仅支持单管理员账号，无普通用户权限划分，适合小型团队 / 个人使用。
✨ 核心功能
1. 会员基础管理
新增会员：录入姓名、联系方式、会员类型、开通时间、有效期等信息；
编辑会员：修改已录入的会员信息，支持更新有效期、联系方式等；
删除会员：移除无效 / 过期的会员记录（谨慎操作，删除后不可恢复）；
列表展示：所有会员信息以列表形式展示，直观查看会员状态。
2. 周期自动管理
有效期计算：根据开通时间 + 周期长度，自动计算会员到期时间；
状态自动标识：系统自动识别「正常」「即将到期」「已过期」会员，不同状态区分显示；
到期提醒：列表中高亮显示即将到期（7 天内）和已过期的会员。
3. 简易查询
支持按会员姓名、联系方式模糊查询，快速定位目标会员；
支持按会员状态（正常 / 过期）筛选，聚焦特定状态的会员。
4. 其他便捷功能
数据本地存储：基于 SQLite3，所有数据存储在群晖本地，无需依赖外部数据库；
轻量化设计：无冗余功能，页面简洁，运行流畅，适配群晖低功耗场景；
图标适配：内置 favicon.ico，浏览器标签页显示系统图标。
⚠️ 注意事项
数据备份：定期备份 member.db（SQLite3 数据库文件），避免数据丢失；
权限设置：建议仅内网访问，群晖 Web 目录权限设置为仅管理员可修改；
PHP 版本：避免使用过高版本 PHP（如 PHP 8.2+），可能存在兼容问题，推荐 7.4~8.0。
总结
本系统是适配群晖 Web Station 的轻量会员周期管理工具，核心依赖 PHP + SQLite3，部署关键是启用 PHP 的 SQLite3 扩展；
核心功能覆盖会员增删改查、周期自动计算、状态标识及简易查询，满足基础管理需求；
部署无需复杂配置，重点完成 PHP 脚本语言中 SQLite3 扩展的启用即可快速使用。
